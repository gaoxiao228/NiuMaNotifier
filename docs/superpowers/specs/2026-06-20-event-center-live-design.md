# 设置页实时事件中心设计

## 背景

设置页目前已有侧边栏面板模型，包含“插件管理”和“通知历史”。主界面也有最近事件摘要，但它服务主状态概览，不适合作为调试窗口查看原始事件。

本设计在设置页新增“事件中心”面板，用于观察 NiumaNotifier 运行期间实时广播的 `NiumaEvent`。用户已明确第一版不显示历史数据，只显示打开事件中心之后收到的实时事件。

## 目标

- 在设置页侧边栏新增“事件中心”入口。
- 事件中心只展示 `NiumaEvent` 原始事件，不展示插件运行事件、配置变更、状态重置等内部 `RuntimeEvent`。
- 只显示实时数据：进入事件中心后连接 `/api/v1/events/stream`，收到事件后追加到底部。
- 事件项默认显示精简摘要，点击后展开格式化 JSON，便于查看完整字段。
- 离开事件中心后关闭 SSE 连接，避免后台长期占用事件流连接。

## 非目标

- 不加载历史事件。
- 不提供“加载更早事件”。
- 不做断线期间事件补偿或 replay。
- 不新增后端分页、cursor 或 `Last-Event-ID` replay 能力。
- 不把事件中心做成完整日志系统，也不提供筛选、搜索或导出功能。

## 可靠性语义

事件中心第一版是实时观察窗口，不是持久化审计日志。

可靠性边界如下：

- 事件中心打开前已经发生的事件不展示。
- SSE 连接成功后收到的 `event` 消息会展示。
- 如果 SSE 断开，断线期间发生的事件不补偿展示。
- SSE 自动重连后，继续展示重连后收到的新事件。
- 前端按事件 `id` 去重，避免浏览器重连或服务端重复发送导致同一事件显示多次。

这个语义与现有 `/api/v1/events/stream` 设计一致：事件流只广播新增且已应用的 `NiumaEvent`，第一阶段不补发历史事件，也不实现 replay。

## 接口约束

本需求不新增业务 JSON 接口。

事件中心使用现有 SSE 协议：

```http
GET /api/v1/events/stream
Accept: text/event-stream
```

SSE 是流式协议例外，响应类型为 `text/event-stream`，不使用统一 JSON 包装。普通业务接口仍遵守项目现有统一响应结构。

前端只处理 `event` 类型消息：

```text
event: event
id: <event.id>
data: <NiumaEvent JSON>
```

控制事件或无法解析的消息不进入事件列表。解析失败时展示连接错误状态，不让单条坏消息破坏已有列表。

## 前端状态

新增事件中心相关状态：

- `activeSettingsPanel` 增加 `event-center`。
- `eventCenterEvents` 保存本次打开事件中心后收到的实时事件。
- `eventCenterExpandedIds` 保存已展开 JSON 的事件 id。
- `eventCenterStream` 保存当前事件中心 SSE 连接。
- `eventCenterStreamConnected` 表示实时连接状态。
- `eventCenterErrorText` 保存连接或解析错误提示。

事件中心面板首次进入时清空 `eventCenterEvents` 和展开状态，然后建立 SSE。切换到其他设置面板或返回主界面时关闭 SSE。再次进入事件中心时重新开始一段新的实时观察会话。

## UI 结构

侧边栏顺序：

1. 插件管理
2. 事件中心
3. 通知历史

事件中心顶部显示标题、简短说明和连接状态：

- “实时已连接”
- “实时连接中”
- “实时已断开”

空态文案为“等待新的实时事件”。收到事件后按到达顺序追加到底部。每条事件的折叠态显示：

- 事件类型
- 工具
- 项目名
- 摘要
- 本地化时间

展开态在摘要下方展示格式化 JSON。JSON 使用等宽字体、保留缩进，并限制最大高度，超出时在块内滚动，避免单条长事件撑坏整个设置页。

## 数据处理

收到 SSE `event` 消息后：

1. 解析 `event.data` 为 `NiumaEvent`。
2. 如果已有相同 `id`，跳过。
3. 追加到 `eventCenterEvents` 末尾。
4. 重新渲染事件中心列表。

格式化 JSON 时使用前端当前 `NiumaEvent` 对象本身，不手写字段列表，避免字段增减时详情区丢失信息。摘要行继续使用已有翻译函数展示事件类型和工具名。

## 异常处理

- SSE `open` 时设置连接状态为已连接，清空错误提示。
- SSE `error` 时设置连接状态为已断开，并显示简短错误提示。
- 浏览器 `EventSource` 会自行尝试重连，前端不额外实现历史补偿。
- 如果单条事件 JSON 解析失败，保留当前列表并显示错误提示。
- 离开面板时关闭 SSE，并清理连接状态，避免重复连接。

## 测试范围

前端测试覆盖：

- 设置页 shell 包含“事件中心”侧边栏入口，默认隐藏事件中心面板。
- 激活 `event-center` 时事件中心面板显示，插件管理和通知历史隐藏。
- 事件中心空态显示“等待新的实时事件”。
- 事件列表按追加顺序渲染，新事件出现在底部。
- 点击事件项后展开格式化 JSON，再次点击收起。
- 响应式 CSS 覆盖事件中心列表、JSON 详情块和连接状态，避免窄屏布局溢出。

实现验证覆盖：

- `npm test`
- `npm run build`

如果实现只改前端，不需要新增 Rust 测试。若后续修改 SSE 后端协议，再补充 `cargo test` 中的事件流测试。

## 后续扩展

后续如果需要“不遗漏历史和断线事件”，应新增后端 replay 能力，例如：

- 支持 `Last-Event-ID` 补发。
- 新增按 `after_id` 或 `since_created_at` 查询事件缺口的接口。
- 为事件中心提供 cursor 分页或持久化审计日志视图。

这些能力不属于第一版实时事件中心。
